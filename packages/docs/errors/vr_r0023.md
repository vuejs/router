# VR_R0023: `next` was never called

> The "next" callback was never called inside of "{name}":
> {guard}
> . If you are returning a value instead of calling "next", make sure to remove the "next" parameter from your function.

**Level:** Warning

Runtime warning logged with `console.warn` during development and stripped from production builds.

## How to fix it

Return the value from the guard instead of calling `next()`, or remove the
`next` parameter.
