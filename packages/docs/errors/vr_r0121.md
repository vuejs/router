# VR_R0121: `history.state` was manually replaced

> history.state seems to have been manually replaced without preserving the necessary values. Make sure to preserve existing history state if you are manually calling history.replaceState.

**Level:** Warning

Runtime warning logged with `console.warn` during development and stripped from production builds.

## How to fix it

Preserve `history.state` when calling `history.replaceState` manually:

```js
history.replaceState(history.state, '', url)
```

See the [migration guide](/guide/migration/#Usage-of-history-state) for more
information.
