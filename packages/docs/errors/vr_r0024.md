# VR_R0024: `next` called more than once

> The "next" callback was called more than once in one navigation guard when going from "{from}" to "{to}". It should be called exactly one time in each navigation guard. This will fail in production.

**Level:** Warning

Runtime warning logged with `console.warn` during development and stripped from production builds.

## How to fix it

Call `next()` exactly once per guard or migrate to returning the value passed to
`next()` instead.
